PVRVFrame Graphical User Interface¶

Working with Profiles¶

Profiles are pre-configured sets of extensions and capabilities. A selection of profiles is included with PVRVFrame. To select a given profile for PVRVFrame, perform the following steps:

  1. Open the PVRVFrame GUI (Fig. 2).

Fig. 2 PVRVFrame GUI

  1. From the Profile dropdown box, select the profile name that matches the device or chipset (Fig. 2 a).
  2. Use the Profile Properties tab to view the capabilities, extensions and EGL configurations for the selected profile (Fig. 3).
  3. Select the Modify option (Fig. 3 a) to be able to add or delete items in the Capabilities, Extensions, and EGL Configs trees. You can also specify your own profile details (Fig. 4).

Fig. 3 Viewing further information

Fig. 4 Editing profile

Note

If no user profiles are displayed in the Profiles dropdown box, click Edit -> Preferences and ensure that the Directory field is set to the directory which contains the profile data files. The default directory for Windows is displayed in Fig. 5. Any user files profiles will be displayed at the top of the list.

Fig. 5 Preferences dialog box

Note

By default, when eglCreateContext is called PVRVFrame checks to see if the host system supports the entire feature set of the requested API, and if it does not, EGL_BAD_ALLOC is reported. However users can change this behaviour by running the GUI, opening the Preferences dialog and enabling ‘Loose’ mode, which allows eglCreateContext to succeed even if the entire feature set isn’t supported.

Running an Application¶

When an OpenGL ES application is run (Fig. 6 a), it automatically connects to PVRVFrame and any GL errors or warnings that occur during the time that the application is run for will be displayed in the Console section of PVRVFrame (Fig. 6 b). By default, a running application will be displayed rendered. An application can also be rendered in wireframe by using the Wireframe checkbox under Rendering Options (Fig. 6 c).

Fig. 6 Running an application

Note

Other actions can also be achieved in PVRVFrame GUI such as changing the font used to display text in the console.
